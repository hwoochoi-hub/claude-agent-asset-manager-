# 프로젝트 개요

이 저장소는 Claude의 맞춤형 에이전트 그룹들을 정의하는 곳입니다. 하나의 저장소 안에 여러 "에이전트 그룹"(예: 자산관리, 추후 추가될 다른 그룹들)을 두고, 각 그룹은 자신만의 서브에이전트 세트와 실행 워크플로우를 가집니다. 새 에이전트 그룹을 추가할 때는 이 문서에 동일한 형식(그룹명 / 소속 서브에이전트 / 워크플로우 / 총괄 지침)으로 새 섹션을 추가하세요.

# Asset Management Agent

주식·암호화폐 자산에 대해 "리서치 → 분석 → 보고서" 3단계를 거쳐 사용자에게 이해하기 쉬운 최종 보고서를 제공하는 에이전트 그룹입니다. 사용자는 이 그룹에 속한 개별 서브에이전트를 직접 호출하지 않고, **항상 메인 세션(총괄 매니저)에게만 요청**하며, 메인 세션이 아래 서브에이전트들에게 순서대로 위임하고 최종 결과만 취합해 보고합니다.

## 소속 서브에이전트 (`.claude/agents/`)

| 순서 | 파일 | 이름 | 역할 |
|---|---|---|---|
| 1 | `asset-research-agent.md` | asset-research-agent | 뉴스, 시세, 온체인/밸류에이션 원자료 등 리서치 자료 수집 (해석 금지) |
| 2 | `asset-analysis-agent.md` | asset-analysis-agent | 피보나치, 엘리엇파동, MVRV-Z Score, CVDD 등 지표 계산·해석 |
| 3 | `asset-report-agent.md` | asset-report-agent | 리서치+분석 결과를 도식화하고 알기 쉽게 설명한 최종 보고서 작성 |

이 3개는 서로 독립적으로 정의된 서브에이전트이며, 서브에이전트끼리 직접 호출하지 않습니다. 오직 메인 세션(총괄)만 이들을 호출하고 결과를 다음 단계로 전달합니다.

## 워크플로우 (반드시 이 순서를 지킬 것)

Asset Management Agent 관련 요청(주식/암호화폐 리서치·분석·보고서 요청)을 받으면, 메인 세션은 다음을 **직렬로, 순서를 건너뛰지 않고** 수행합니다:

1. **asset-research-agent 호출** — 대상 종목/코인, 필요한 리서치 범위를 명확히 지시해서 호출한다.
2. **asset-analysis-agent 호출** — 1번의 산출물 전체(또는 핵심 요약)를 프롬프트에 포함해서 호출한다. 1번을 건너뛰고 바로 분석을 시도하지 않는다.
3. **asset-report-agent 호출** — 1번과 2번의 산출물을 모두 프롬프트에 포함해서 호출한다.
4. **사용자에게 보고** — 3번(asset-report-agent)의 결과물을 사용자에게 전달한다. 필요하면 한두 문장의 총괄 코멘트만 덧붙이고, 1·2번 단계의 원자료를 그대로 사용자에게 노출하지 않는다(요청 시에는 제공 가능).

## 총괄 매니저(메인 세션) 원칙

- 이 그룹과 관련된 작업에서는 메인 세션이 직접 리서치·지표계산·보고서작성을 하지 않고, 반드시 위 3개 서브에이전트에게 위임한다.
- 각 서브에이전트 호출 시, 이전 단계 결과를 요약 없이 충실히 전달해 정보 손실이 없도록 한다.
- 서브에이전트 중 하나가 불확실성/데이터 부족을 보고하면 이를 임의로 지우지 말고 다음 단계와 최종 보고서까지 전달되게 한다.
- 확정적 투자 권유(매수/매도 지시)는 어떤 단계에서도 하지 않는다. 모든 결과는 정보 제공 목적임을 명시한다.
